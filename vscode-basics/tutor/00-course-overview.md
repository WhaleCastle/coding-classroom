# VS Code Basics — Overview (for the tutor)

A SHORT starter course (3 chapters, roughly one session each) that makes the
student comfortable in VS Code **before** any coding. Do this course first.

No code is written in this course except one tiny taster at the very end.
Everything is about the tool: files, editing, and the terminal.

## Curriculum map

| # | Chapter | Core ideas | Mini-product |
|---|---------|-----------|--------------|
| 1 | Meet VS Code | the 4 areas (Explorer, Editor, Terminal, Chat), open a folder, create/rename/delete files & folders, save | His own tidy workspace folder tree |
| 2 | Editing Like a Pro | tabs, the unsaved dot, undo/redo, Find, select & move text, zoom | A fixed-up messy text file |
| 3 | The Terminal | what a terminal is, where am I (cd / ls / pwd), run a program, stop a program, clear | Navigate anywhere + run his first one-line Python file |

## Rules for this course

- Short sessions. Each chapter ≈ 20–30 minutes.
- Everything is hands-on: never explain two things in a row without the student
  DOING something in between.
- The keyboard shortcuts taught are the survival set only: Save, Undo, Find,
  Copy/Paste. Do not flood him with shortcuts.
- OS note: ask ONCE at the start of Chapter 1 whether he is on Windows or Mac,
  remember it (note it in the progress block), and give the right keys
  (Ctrl vs Cmd) and terminal commands (dir vs ls work differently; in the
  VS Code default terminal on Windows PowerShell, `ls` and `pwd` also work).
